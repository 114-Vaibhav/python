# Custom ORM (Object-Relational Mapper)

This project is a simple custom ORM built in Python using `sqlite3`. It demonstrates how ORM concepts work internally by mapping Python classes to database tables without using frameworks like SQLAlchemy or Django ORM.

The project supports:

- model definition using Python classes
- automatic table creation
- field types like `CharField` and `IntegerField`
- `ForeignKey` relationships
- inserting records with `save()`
- deleting records with `delete()`
- filtering and ordering records
- reverse relation access using `related_name`

## Project Files

- `main.py`  
  Contains the complete ORM implementation and usage example.

- `orm.db`  
  SQLite database created by the project.

- `Output.txt`  
  Contains the terminal output showing SQL queries, record creation, filtering, related data access, and deletion.

## How It Works

This ORM is built step by step using custom Python classes:

### `Field`

The base field class stores metadata such as:

- SQL column type
- nullable option
- unique option

### `CharField` and `IntegerField`

These are custom field types used to define model attributes.

### `ForeignKey`

Used to create relationships between models. In this project, `Post` is linked to `User`.

### `ModelMeta`

This metaclass collects all model fields automatically and generates:

- `_fields`
- table name from class name

### `Model`

The base model provides:

- `create_table()`
- `save()`
- `delete()`
- `filter()`
- reverse relation lookup through `__getattr__`

### `QuerySet`

Used for querying records with:

- `filter(age__gte=25)`
- `order_by("-name")`
- `all()`

## Models Used

### `User`

Fields:

- `name`
- `email`
- `age`

### `Post`

Fields:

- `title`
- `author` as a foreign key to `User`

## Example Operations in This Project

The code in `main.py` demonstrates:

1. creating `user` and `post` tables
2. inserting three users
3. inserting three posts
4. filtering users with age greater than or equal to 25
5. ordering results by name in descending order
6. fetching all posts for a specific user using reverse relation
7. deleting one post
8. deleting one user

## Example SQL Generated

The project prints SQL statements in the terminal, such as:

```sql
CREATE TABLE IF NOT EXISTS user (...)
INSERT INTO user (name, email, age) VALUES (?, ?, ?)
SELECT * FROM user WHERE age >= 25 ORDER BY name DESC;
DELETE FROM post WHERE id = ?
```

This helps in understanding how ORM methods are translated into SQL queries.
